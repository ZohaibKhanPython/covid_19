What we know so far- We have so far understood this is a high dimentional dataset with 19476 dataset - We have found that theres no nan values , We have converted all features that were object type to float and dropped the features which had no use for our method.

Our Issue - how to reduce these features - 19476 is massive amount of features our model has to go through which could be heavy for laptops/ computers would take time to run. Also could effect the accuracy of our model.

There is a commonly used method PCA-Principal Component Analysis (is an unsupervised, non-parametric statistical technique primarily used for dimensionality reduction in machine learning.) which is normally used to reduced the amount of features- the idea is bring down 19476 to under 100 features

After playing around with the number of features i want to keep for my X axis ( independent variables ), i decided to ask pca to reduce features down to 53 features to go into the model , you will see that i have put 53 n_components ( i asking pca on how many features i want it to down to).

I have then transformed the independent variable (X) with the reduced amount of features- the reason for this doing is to make sure our method does not hang with computing time and it is more efficient.

random_grid = { 'n_estimators': [100,200,300,400,500],>>This is the number of trees in the forest- the higher the number the better variance. , i kept it to maximum 500, it also increases the computing time as the number trees increases this is why i kept it to maximum 500 so it does not exhaust the pc. 'max_features': randint(1,53),>>Maximum features are 53 because thats we set by using pca function - this will be selecting number of features to consider in our method 'max_depth': [3,5,10,32,None], the maximum depth can be upto 32 , but i have decided to keep the maximum 32 (this is how deep can go). The deeper the tree, the more splits it has and it captures more information about the data. i have tried different variation.

'criterion':['gini','entropy'], This parameter is the function used to measure the quality of a split and it allows users to choose between ‘gini‘ or ‘entropy, I am using criterion to ensure that features when splitted are have a tidy splits and data is not messy. I have selected both criterion so when i choose the param it can decide which one is best to use.

'bootstrap':[True,False], 'min_samples_leaf':randint(1,4),

} The theory shows that the accuracy of random forest can be decomposed as:

            bias2+1kvariance+(1−1k)covariance+σ2

max_depth represents the depth of each tree in the forest. The deeper the tree, the more splits it has and it captures more information about the data. We fit each decision tree with depths ranging from 1 to 32 and plot the training and test errors

After selecting my parameters, I have then decided use these parameters in hypertuning method i created , In which i am using this paramters above (random_grid), In this function i have used randomisedSearchcv method to run through the parameters above.

Why am i using RandomisedSearchCV cross validation is simply because it is helping me choose the Best parameters that would suite my random forest method to try to get the best results. i.e it would tell me how many trees should my method have in n_estimatores , how deep the tree needs to be (depth)..etc , it would tell me which criterion works best with this dataset...etc. My idea is run this method to get the Best param and recall it using ht_params which i have defined to return, after running this method it should print the best parameters and numbers associated with, with this information i would run random forest again.

To make this work the block of code below is simply using the the object rfc where i have created randomforestclassifier in randomisedsearchcv. it is using the parameters in random_grid{} and it would keep iterating through it and it has fully run the parameters,
once it has done that, i can then recall ht_params for the best parameters to use in my randomforest method. to cut out computing time i have use PCA first to reduced the amount of features. it would run 5 folds with 33 candidates.

To this purpose:

The higher the value of n_estimators the better the variance term is reduced. However, also higher the computational time. A low value of max_features generates biased and uncorrelated trees, therefore the final model may contain some bias. An high value of max_features generates unbiased but correlated trees, therefore the final model may contain some variance. Conversely, a fine tuning may be needed to find the optimal value of max_features.

The code below does so.

##Regression Model using RandomizedSearchCV 

from sklearn.model_selection import RandomizedSearchCV
from scipy.stats import randint

rfc1 = RandomForestClassifier(n_jobs=-1)
rfc1.fit(X_train,y_train.ravel())

# create random grid
random_grid = {
 'n_estimators': [100,200,300,400,500],
 'max_features': randint(1,53),
 'max_depth': [3,5,10,32,None],
 'criterion':['gini','entropy'],
 'bootstrap':[True,False],
 'min_samples_leaf':randint(1,4),
 
 }
def hypertuning_rscv(rfc1,random_grid,nbr_iter,X,y):
    rfc_random = RandomizedSearchCV(estimator = rfc1, param_distributions = random_grid, n_iter = nbr_iter, cv = 5, verbose=2, random_state=42, n_jobs = -1)
    rfc_random.fit(X,y)
    ht_params =rfc_random.best_params_
    ht_score =rfc_random.best_score_
    return ht_params,ht_score

y_pred= rfc1.predict(X_test)

rf_parameters, rf_ht_score = hypertuning_rscv(rfc1,random_grid,50,X,y.values.ravel())
